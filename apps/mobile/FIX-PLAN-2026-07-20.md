# Fix plan — audit 2026-07-20

Companion to `AUDIT-2026-07-20.md`. One section per finding, in execution order.
Ordering is dependency- and risk-driven: honesty/money first, then auth, then
surface truthfulness, then cosmetics.

**Guiding principle:** never ship a surface that asserts something untrue. Where
a claim can be made *true* cheaply, make it true. Where it cannot, either remove
the surface or label it unmistakably. Disabling with a stated reason beats a
control that silently does nothing.

**Definition of done for every item:** `tsc --noEmit` clean · jest green ·
smoke-tested on the headless `@audric_test` AVD against a real signed-in session.

---

## Phase 1 — kill the fabricated balance (CRITICAL)

Two halves: remove the lie, then make the honest answer possible.

### 1a. Delete the wallet demo path

- `app-state/store.tsx:218` — drop the `wallet` branch from `classify()`. Wallet
  wording then classifies as `"text"` and reaches the real model.
- `app-state/store.tsx:226-277` — drop the `wallet` case from `buildDemoReply()`
  and narrow its parameter type to `"image" | "video" | "artifact"`.
- `components/chat/conversation.tsx:183-213` — delete `WalletCard` (the
  hardcoded `124.50` / `0.82`).
- `components/chat/conversation.tsx:148-181` — delete `WorklogCard` (hardcoded
  "Worked for 8s · 3 steps") and `WORK_STEPS` in `catalog.ts:123-127`. It only
  ever rendered for the wallet demo turn.
- `components/chat/conversation.tsx:119-120` — drop both `demo === "wallet"` render branches.

### 1b. Make "what's my balance" actually work

Add a real `balance_check` tool to `app/api/chat+api.ts`, reusing exactly what
`app/api/balance+api.ts` already does (`SuiGrpcClient` + `queryBalance` from
`@t2000/sdk`, network from `EXPO_PUBLIC_SUI_NETWORK`).

- Address comes from the **verified session** (`auth.userId`), never from the
  model or the request body — same rule the data routes follow.
- Guest / untokened turns: the tool reports no connected wallet rather than
  guessing.
- Register alongside `web_search` in the `streamText` `tools` map so the
  existing Chain-of-Thought timeline renders the call as a real tool step.

This converts the app's own first suggestion chip from a fabrication into a
truthful answer, and removes the need for any wallet demo card.

### 1c. Label the remaining demo surfaces

Image / video / artifact demo replies are the same class of problem (they render
as if the model generated content) but carry no financial claim, and they retain
genuine prototype value. Rather than deleting them, make them unmistakable: a
small `Demo` badge on `ImageCard`, `VideoCard`, and the artifact card in
`conversation.tsx`, styled as a muted chip. No behavior change.

## Phase 2 — enforce session expiry (HIGH)

`auth/useAuth.tsx:58-68`. In the launch effect, treat a past `expiresAt` as no
session:

- If `s?.expiresAt` is set and `Date.now() >= s.expiresAt` → `clearSession()` +
  `clearWalletKeys()`, land on the gate.
- A session with **no** `expiresAt` is an untokened guest session, not an expired
  one — leave it alone (matches `session.ts:15`'s contract).
- Clear the wallet keys alongside the session so a stale key set can never
  outlive the identity that authorised it — same invariant `signIn` already
  enforces on its no-fresh-proof path.

Unit-test the predicate rather than the effect: expired → cleared, future →
kept, absent → kept.

## Regression caught and fixed during the run

The first `app/callback.tsx` was a bare spinner. `RootNavigator` routes between
`(app)` and `gate` via `Stack.Protected`, and `callback` is in **neither** group — so
the deep link landed there and nothing ever moved the user along. Sign-in hung on a
spinner. Fixed by redirecting to `/` once `status` settles, handing control back to the
guarded stack. Verified: OAuth now completes into onboarding.

## Check #1 closed — 2026-07-21

A working `AI_GATEWAY_API_KEY` arrived, so the last hop — *does the model actually call
the tool, and does the app render what it returns* — was run on-device for real.

Tapped **"Check my balance"** on the same signed-in address. The reply:

> **Thought for 14s · 1 step**
> Your USDC balance is **0 USDC**.
> You currently hold **0.846 SUI** (~$0.85) in your wallet on testnet.

Expanding the Chain-of-Thought step shows the tool's own return values, not prose the
model composed:

```
Connected: true
Address:   0xcc4e70138b3f5c1c086a89abfdab2c67a31c2f190709ee46549fd2cd9282c0a1
Network:   testnet
USDC:      0
SUI:       0.84600424
Total USD: $0.84600424 (from SUI holdings)
```

`0.84600424` is the same on-chain figure the wallet tab renders as `0.85`, and the
address is the one **bound at tool construction** — the model never chose it. The
CRITICAL finding is closed end-to-end: no `124.50`, no invented figure, real chain data
reaching the screen through `balance_check`.

Kept for the record, the earlier out-of-band proof (tool module driven straight against
the chain, no model, no gateway) — it also covers the guest branch the on-device run
cannot reach while signed in:

```
guest -> {"connected":false,"message":"No wallet is connected — …"}
real  -> {"connected":true,"network":"testnet","usdc":0,
          "sui":0.84600424,"totalUsd":0.84600424}
```
