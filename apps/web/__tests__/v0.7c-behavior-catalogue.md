## Status legend

- **Phase 0 baseline** — captured at Phase 0 acceptance (G1 gate). Each behavior gets a one-line description of "what production currently does" against the chat shell as-of `apps/web@HEAD` 2026-05-18.
- **Phase 6 acceptance** — verified post-cutover (G9 gate). Each behavior gets a yes/no/regression-note entry.
- **Disposition** — for each behavior, the v0.7c migration shape:
  - **STREAM** — heavy migration (consumes custom SSE events; full rewrite to AI SDK parts via `experimental_providerMetadata`)
  - **PROP** — light migration (consumes typed tool results; mechanical prop-source swap)
  - **CHANNEL** — channel change only (canvas HTML / MPP response stays the same shape)
  - **UNCHANGED** — server-side cron / silent-intelligence work that doesn't touch the chat-shell wire format
  - **NEW** — net-new behavior unlocked by the fork (Slice D, generative UI, etc.)
